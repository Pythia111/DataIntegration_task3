3.5   基于   XML   数据集成的 集成教务系统 示例  近年来教育信息量的不断增多， 使各级各类学校、 家庭和教育管理部门对教育信息计算 机和教育信息服务的要求越来越强烈。 基于计算机在教育信息管理和信息服务显著的辅助作 用，校园信息化在学校教育管理中日益普及。  原有三个独立的院系选课系统，地理上分散、结构上异质的。系统结构图如下图所示：  A 客户端  B 客户端  C 客户端  C 服务器  B 服务器  A 服务器  Sqlserver Oracle Mysql  图   3-5 ：原教务系统系统结构图  原来的三个系统采用   C/S   结构或   B/S   结构， 但选择不同的数据库以及不同的数据表结构 和具体的程序逻辑结构各自独立实现了选课系统。  本项目基于原有的院系教务系统应用异构数据库集成技术， 给出具有跨系选课功能的集 成教务系统实例。 集成教务系统项目通过将不同院系数据库的异构数据进行数据集成， 分析 和处理， 使得用户在访问和处理不同院系的异构数据库中数据的同时， 不必知道物理数据库 的分布和结构，只需要类似本地查询和管理的操作。  使用数据集成技术将使系统具有以下特点：  1)   透明性  整个集成系统对客户端屏蔽了通信的细节， 使得源系统在绝大部分不需要改动， 保留各 自系统的风格和使用方式。  2)   可扩展性  采用事实上通信格式的圭臬   XML   作为通信数据的标准，使得集成之后的新系统更容易 地去集成未来可能的其他教务系统，而且更大程度的复用已有的功能与代码。  3)   健壮性  XML   有许多成熟的产品、技术和应用，对已有的具有良好的性能的   XML   产品或开源 成果的应用， 系统将会得到更好的健壮性和良好的性能， 并使得开发人员把注意力集中在业 务逻辑的领域范畴内。

4)   安全性  在分布式的环境下，各个系统是相对独立的。它们的各自系统独立运作，保留各自的关 键数据和信息， 只暴露给外界需要暴露的数据。 同时集成服务器也降低了各个系统服务器的 访问负担。  3.5.1   原有教务系统分析  1.   原系统用例分析  学校教务系统 学生 管理员 登录验证 个人信息管理 统计信息查看 课程信息管理 教务信息管理 后台管理  图   3-6 ：原教务系统用例图  对单个院系教务系统，用户可以包括学生，系统管理员，用户需求如下：  1)   对学生用户：  ①   登陆验证：对不同年级的学生设置不同的选课权限  ②   个人信息管理：个人信息查询更新  ③   个人课程管理：课程查看，选择，删除和更新  2)   对管理员用户：  ①   登陆验证：对不同级别的管理员设置相应的管理权限  ②   个人信息管理：个人信息查询更新  ③   统计信息查看：查看本院系学生、课程、课程被选情况等统计信息  ④   教务信息管理：对课程，开放课程，学生，选课等数据的设置操作  ⑤   后台管理：增删管理员，启动关闭服务器，查看在线情况等等  2.   原系统的异构性 分析  1)   现有院系   A 、院系   B   和院系   C   的教学管理系统基于不同的数据库系统。其中   A   的 数据库系统使用   SqlServer ， B   的数据库系统使用   Oracle ， C   的数据库系统使用   MySQL   实现； 其中每个数据库系统都包含了学生、课程以及选课的信息。
