## 2. XML报文请求与响应示例

我们在跨院调用（请求共享课程、发起跨校选课/退课）等交互场景下，均以学院A本地定义的 XML 格式进行数据发送/接收。

### 场景1：集成服务器请求学院A目前的“共享课程”列表（响应）
当集成服务器收集跨院共享课程时，学院A将给出以下格式的XML：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Classes>
  <class>
    <课程编号>C_A001</课程编号>
    <课程名称>数据结构</课程名称>
    <学分>4</学分>
    <授课老师>张老师</授课老师>
    <授课地点>教A-101</授课地点>
    <共享>Y</共享>
  </class>
  <!-- 更多共享课程... -->
</Classes>
```

### 场景2：学生发起的跨校选课请求（请求）
当本地学生想要选择学院B或C的课程时，将从学院A本地发送以下结构给集成服务器处理：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<choiceReq>
  <traceId>a231bcd...</traceId>
  <sid>A2026001</sid>
  <cid>C_B001</cid>
  <operation>ENROLL</operation> <!-- 退课为 DROP -->
</choiceReq>
```

集成服务器在处理目标校验和写入后，会返回对应的回执响应，学院A按规约接手解析即可判断选/退课状态。

## 3. 部署与运行说明

本项目采用 Docker 容器化部署 SQL Server 数据库，并使用 Maven 管理 Java 客户端运行环境。以下是详细的部署与运行步骤：

### 3.1 数据库环境部署 (Docker)
1. **启动容器**：请确保电脑运行了 Docker 环境，然后拉取并启动 SQL Server 2022 容器：
   ```powershell
   docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=YourStrongPassword123" -p 1433:1433 --name sql_server_a -d mcr.microsoft.com/mssql/server:2022-latest
   ```
2. **初始化数据**：待容器完全启动后，将包含宽字符（`NVARCHAR`）支持的建立表单和模拟数据导入容器并执行：
   ```powershell
   docker cp member2/sql/db_init_A.sql sql_server_a:/tmp/db_init_A.sql
   docker exec sql_server_a /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P 'YourStrongPassword123' -No -C -i /tmp/db_init_A.sql
   ```
   *经过此步，数据库配置完毕，数据和表结构可保证中文不会乱码，重启客户端也依然保留改变。*

### 3.2 启动 Java 客户端与服务
请在终端中进入 `member2` 文件夹目录，直接执行一键编译与启动命令：
```powershell
mvn clean compile exec:java
```
执行后：
1. 本地的 **Swing GUI 窗口程序** 会弹出以供操作。
2. 程序后台会自动开启 **HTTP Server** 并监听 **8081端口**，以响应集成服务器随时对 `/api/local/sharedCourses` 的数据拉取。
