# 学院B（成员3）交付与说明文档

本目录为“学院B教务系统（Oracle + GUI + 登录 + 本地业务）”的完整交付物。

## 1. 数据库与初始数据

- 数据库：Oracle
- 初始化脚本：member3/sql/init_B.sql
- 表：AccountB、StudentB、CourseB、CourseChoiceB
- 数据量：50名学生（B001~B050）、10门课程（B101~B110）、每生5门课（共250条选课）、10个账户（含管理员）

### 登录账号示例

- 管理员：账号 `admin`，密码 `admin123`
- 学生：账号 `B001`，密码 `b00100`（B002 对应 `b00200`，以此类推）

## 2. 通信接口（本地HTTP服务）

本系统启动后会开启学院B本地HTTP服务，端口为 **8082**，用于被集成服务器调用：

- `GET /api/local/sharedCourses`：导出本院共享课程（XML，学院B本地格式）
- `POST /api/local/enroll`：处理跨院选课落库（请求体XML中解析 `sid`、`cid`）
- `POST /api/local/drop`：处理跨院退选落库（请求体XML中解析 `sid`、`cid`）
- `GET /api/local/statistics`：返回本院学生/课程/选课统计（XML）

## 3. 与集成服务器交互

学院B客户端侧默认访问集成服务器：`http://localhost:8080`：

- `GET /api/integration/sharedCourses?source=B`
- `POST /api/integration/courseChoice?source=B`
- `GET /api/integration/statistics`

若集成服务器尚未实现/未启动，跨院共享课程与跨院选退功能会提示连接失败，但本院选课/退课与课程共享标记修改可正常使用。

## 4. 运行方式

1) 先在 Oracle 中执行初始化脚本 member3/sql/init_B.sql（在你的 schema 下执行即可）

2) 进入 member3 目录运行：

```powershell
mvn clean compile exec:java
```

3) 如无法连接数据库，请修改 `com.collegeB.util.DatabaseConnection` 里的 JDBC URL/用户名/密码。
