# 集成服务器部署与接口文档

## 一、项目概述

集成服务器是基于XML的异构教务数据集成系统的核心中间件，负责：
- 接收三个学院（A/B/C）的HTTP请求并路由到对应处理器
- 对收到的XML进行XSD结构验证
- 使用XSLT完成各学院本地XML格式与统一XML格式之间的转换
- 实现课程共享、跨院选课、跨院退选课的转发逻辑
- 提供全局统计信息聚合

## 二、部署说明

### 2.1 环境要求
- JDK 8 或以上
- Maven 3.6+
- 三个学院本地服务已启动（端口8081/8082/8083）

### 2.2 编译
```bash
cd member5
mvn compile
```

### 2.3 启动
```bash
mvn exec:java
```
或指定端口：
```bash
mvn exec:java -Dexec.args="9090"
```
默认监听端口：**8080**

### 2.4 验证启动
控制台输出：
```
集成服务器已启动，监听端口 8080 ...
接口列表:
  GET  /api/integration/sharedCourses?source={A|B|C}  - 获取其他学院共享课程
  POST /api/integration/courseChoice?source={A|B|C}    - 跨院选课/退选课
  GET  /api/integration/statistics                    - 全局统计信息
```

## 三、接口定义

### 3.1 获取其他学院共享课程

- **URL**: `GET /api/integration/sharedCourses?source={A|B|C}`
- **参数**: `source` — 请求方学院标识（A/B/C）
- **Content-Type**: `application/xml; charset=UTF-8`

**处理流程**：
1. 根据 `source` 确定请求学院
2. 向另外两个学院的 `/api/local/sharedCourses` 发起HTTP GET请求
3. 对收到的本地课程XML进行XSD验证
4. 使用XSLT将各学院本地课程XML转换为统一格式
5. 合并统一格式的课程数据
6. 使用XSLT将合并后的统一格式转换为请求学院的本地格式
7. 返回转换后的课程XML

**请求示例**：
```
GET /api/integration/sharedCourses?source=A
```

**响应示例**（学院A格式）：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Classes>
  <class>
    <课程编号>B101</课程编号>
    <课程名称>高等数学</课程名称>
    <学分>4</学分>
    <授课老师>张老师</授课老师>
    <授课地点>教B-101</授课地点>
    <共享>Y</共享>
  </class>
  <!-- 更多课程... -->
</Classes>
```

### 3.2 跨院选课/退选课

- **URL**: `POST /api/integration/courseChoice?source={A|B|C}`
- **参数**: `source` — 请求方学院标识（A/B/C）
- **Content-Type**: 请求 `application/xml`，响应 `application/xml`

**请求体格式**（choiceReq XSD）：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<choiceReq>
  <traceId>uuid-xxx</traceId>
  <source>A</source>
  <sid>A2026001</sid>
  <cid>B101</cid>
  <operation>ENROLL</operation>
</choiceReq>
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| traceId | string | 请求追踪ID |
| source | A/B/C | 请求来源学院 |
| sid | string | 学生学号 |
| cid | string | 目标课程编号 |
| operation | ENROLL/DROP | 操作类型：选课或退选 |

**处理流程**：
1. 解析请求XML，提取 `sid`、`cid`、`operation`
2. 根据课程编号前缀判断目标学院（B开头→学院B, C开头→学院C, C_A/A开头→学院A）
3. 构建本地选课/退选请求XML，转发至目标学院的 `/api/local/enroll` 或 `/api/local/drop`
4. 接收目标学院的响应并返回给请求方

**响应格式**：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<response>
  <status>SUCCESS</status>
  <message>选课成功</message>
</response>
```

### 3.3 全局统计信息

- **URL**: `GET /api/integration/statistics`
- **Content-Type**: `application/xml; charset=UTF-8`

**处理流程**：
1. 分别向学院A/B/C的 `/api/local/statistics` 发起GET请求
2. 聚合三个学院的统计信息
3. 返回合并后的统计XML

**响应示例**：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<statistics>
  <college name="A">
    <students>50</students>
    <courses>10</courses>
    <choices>250</choices>
  </college>
  <college name="B">
    <students>50</students>
    <courses>10</courses>
    <choices>250</choices>
  </college>
  <college name="C">
    <students>50</students>
    <courses>10</courses>
    <choices>250</choices>
  </college>
</statistics>
```
