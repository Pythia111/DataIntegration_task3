## 三、全局统计模块

### 3.1 接口定义

- **URL**: `GET /api/integration/statistics`
- **Content-Type**: `application/xml; charset=UTF-8`

### 3.2 返回示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<statistics>
  <college name="A">
    <students>50</students>
    <courses>10</courses>
    <choices>250</choices>
  </college>
  <college name="B">
    <students>50</students>
    <courses>10</courses>
    <choices>252</choices>
  </college>
  <college name="C">
    <students>50</students>
    <courses>10</courses>
    <choices>250</choices>
  </college>
  <summary>
    <totalStudents>150</totalStudents>
    <totalCourses>30</totalCourses>
    <totalChoices>752</totalChoices>
  </summary>
  <timestamp>2026-05-13T10:30:00</timestamp>
</statistics>
```

### 3.3 特性

- 分别查询三个学院的学生数、课程数、选课数
- 自动计算总计（summary）
- 包含统计生成时间戳
- 对每个学院的统计结果进行容错处理（单个学院失败不影响整体）
- 通过 `statistics.xsd` 进行XML结构校验

## 四、退选课完整流程

### 4.1 处理流程

```
学生在源学院发起退选（跨院课程）
  → 源学院构造退选XML请求
  → 源学院发送至集成服务器 POST /api/integration/courseChoice?source=X
  → 集成服务器解析请求，提取sid/cid/operation
  → 根据课程编号前缀判断目标学院
  → 校验：不能退选本院课程、operation必须为DROP
  → 转发至目标学院 POST /api/local/drop
  → 目标学院验证选课记录是否存在
      → 存在：删除记录，返回SUCCESS
      → 不存在：返回FAIL + BIZ-004（退选目标记录不存在）
  → 集成服务器将结果返回源学院
```

### 4.2 错误码体系

| 错误码 | 含义 | 触发场景 |
|--------|------|----------|
| BIZ-001 | 重复选课 | 学生已选该课程 |
| BIZ-002 | 课程未开放共享 | 课程share=N |
| BIZ-003 | 课程不存在 | 课程编号无效 |
| BIZ-004 | 退选目标记录不存在 | 退选未选过的课程 |
| SYS-001 | 下游调用超时 | 目标学院无响应 |
| SYS-002 | XML转换失败 | XSLT异常 |
| SYS-003 | 未知系统异常 | 其他异常 |
| VAL-001 | 缺少必填字段 | 请求XML不完整 |
| VAL-002 | 枚举值非法 | operation不是ENROLL/DROP |

## 五、异常处理与容错

### 5.1 超时重试

`HttpUtil.java` 提供 `httpGetWithRetry` 和 `httpPostXmlWithRetry` 方法：
- 支持配置最大重试次数和重试延迟
- 对 `SocketTimeoutException` 单独处理
- 每次重试记录详细日志

### 5.2 日志规范

`LogUtil.java` 提供统一日志格式：
```
2026-05-13 10:30:00.123 [INFO] [com.integration.handler.EnhancedDropHandler] Drop succeeded: traceId=xxx, student=A2026001, course=B101
```

## 七、与成员5的集成说明

成员6的代码是成员5集成服务器的**增量增强**，集成方式：

1. **替换 StatisticsHandler**: 将 `EnhancedStatisticsHandler` 替换原有 `StatisticsHandler`，获得带summary和timestamp的完整统计
2. **增强 Drop 处理**: `EnhancedDropHandler` 提供更完善的退选课逻辑，包含完整的错误码体系
3. **替换 HttpUtil**: 使用带重试机制的版本，提高系统健壮性
4. **添加统计XSD**: `statistics.xsd` 是新增的验证文件

在成员5的 `IntegrationServer.java` 中替换注册：
```java
// server.createContext("/api/integration/statistics", new StatisticsHandler());
server.createContext("/api/integration/statistics", new EnhancedStatisticsHandler());
```

## 八、联调环境配置指南

### 8.2 启动顺序

**必须严格按以下顺序启动各服务：**

| 步骤 | 服务 | 命令 | 端口 |
|------|------|------|------|
| 1 | 学院A本地服务器 | `cd member2 && mvn exec:java` | 8081 |
| 2 | 学院B本地服务器 | `cd member3 && mvn exec:java` | 8082 |
| 3 | 学院C本地服务器 | `cd member4 && mvn exec:java` | 8083 |
| 4 | 集成服务器(核心) | `cd member5 && mvn exec:java` | 8080 |
| 5 | 扩展功能测试 | `cd member6 && mvn exec:java` | 9090 |

### 8.3 各学院本地接口要求

集成服务器依赖各学院提供以下本地接口：

| 接口路径 | 方法 | 用途 |
|----------|------|------|
| `/api/local/sharedCourses` | GET | 返回本学院共享课程XML |
| `/api/local/enroll` | POST | 处理选课请求 |
| `/api/local/drop` | POST | 处理退选请求 |
| `/api/local/statistics` | GET | 返回本学院统计数据 |

### 8.4 联调验证清单

| 验证项 | 方法 | 预期结果 |
|--------|------|---------|
| 课程共享 | `GET http://localhost:8080/api/integration/sharedCourses?source=A` | 返回B和C的共享课程（已转换为A格式） |
| 跨院选课 | `POST http://localhost:8080/api/integration/courseChoice?source=A` | 返回SUCCESS，B或C数据库新增记录 |
| 跨院退选 | `POST http://localhost:8080/api/integration/courseChoice?source=A` (operation=DROP) | 返回SUCCESS，对应记录删除 |
| 全局统计 | `GET http://localhost:8080/api/integration/statistics` | 返回A/B/C统计数据+summary+timestamp |
| 重复选课 | 对同一学生同一课程再次ENROLL | 返回FAIL + BIZ-001 |
| 退选不存在 | 对未选课程执行DROP | 返回FAIL + BIZ-004 |

### 8.5 常见问题排查

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| Connection refused | 目标学院服务未启动 | 检查对应端口是否在监听 |
| XSD validation failed | XML格式不符合Schema | 检查XML字段名、枚举值是否符合XSD定义 |
| 编码乱码 | 未统一使用UTF-8 | 确保所有XML声明encoding="UTF-8" |
| 统计数据为0 | 数据库未初始化 | 运行各学院的SQL初始化脚本 |
| Read timeout | 学院服务响应过慢 | 增加超时时间或检查数据库连接 |
